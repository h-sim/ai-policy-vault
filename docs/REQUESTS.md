# AI Policy Vault — 監視リクエスト（面談なし）

このページは **「監視してほしいソース」や「欲しい通知の形」** を、面談なしで受け付けるためのテンプレです。
※本サービスは **変化の検知（Detection）** のみを行い、内容の解釈・法務判断（Judgment）は行いません。

---

## 1. 何ができる？（MVPの前提）

- 規約・仕様・リリースノート等の「変化」を定期実行で検知します
- 影響度（Breaking / High / Medium / Low）に分類し、優先確認対象を絞ります
- 証跡（snapshot hash / item ID など）を残します

---

**タイトル案**: 監視リクエスト：<サービス名 / ドキュメント名>

- 監視したいURL（複数可）:
  -
  -

---

## 3. 注意事項（重要）

- 「変更未検出」は「変化なし」を意味しません。正式な扱いは **未検出（要目視確認）** です。
- 変化内容の正確性・完全性は保証しません。一次情報（公式原文）で目視確認してください。
- 本サービスは Detection のみ。解釈・リスク評価（Judgment）は人間が実施します。

---

## 4. 送付方法（GitHub Issue）

### 受付窓口（面談なし）

- 受付は **GitHub Issues** で行います（無料・履歴が残るため）。
- 依頼はこのリポジトリの Issues に起票してください：
  - https://github.com/h-sim/ai-policy-vault/issues

> メールやフォームは使いません。やり取りは Issue 上で完結します。

### 手順

1. リポジトリの「Issues」を開く
2. 「New issue」を押す
3. 上のテンプレを貼って送信

---

## 5. 非対応（MVPでは対象外の例）

- ToSページ等の「HTML監視の完全なノイズ除去」（必要なら別途 normalizer 実装）
- 「見逃しゼロ」の保証
- 法務判断・リスク評価の代行
